# 审查拉取请求

## 用户请求

“请在合并前审查这个订单查询改动，重点看权限与旧客户端兼容性。”

## 工作流

1. 合并 `config/defaults.yaml` 与项目根目录的 `hogen-codex.yaml`，确认审查基线和待审文件。
2. 阅读 diff、调用点和接口契约，检查权限、空值、旧参数兼容性与相关测试。
3. 依 P0–P3 记录每个可证实发现，并写入证据、影响和最小建议。
4. 若没有 PR/CI 工具，只使用 Git、代码阅读、`rg` 和已运行的本地测试，并报告未覆盖项。

## 最终报告

发现 P1：新门店过滤条件没有应用到导出接口，拥有导出权限的用户可看到超出门店范围的数据；证据为导出路径未传递门店条件。建议复用查询约束并增加跨门店导出测试。未验证远端 CI，因为当前会话没有对应访问能力。
