# 记录架构决策

## 用户请求

“请把本项目的通知重排必须经由共享 orchestrator、不要放进 SwiftUI View 的决定保存下来。”

## 工作流

1. 读取默认配置与项目 `hogen-codex.yaml`，以项目值覆盖默认值。
2. 检测记忆后端的检索与写入能力，先检索是否已有相同决策。
3. 将已确认的约束、替代方案和验证依据按决策模板写入指定项目；若没有写入能力，仅报告缺失并请用户确认人工记录位置。

## 最终报告

已记录：通知重排由共享 orchestrator 负责，SwiftUI View 仅调用该服务。该结论来自用户确认；后续修改需保持此边界。若后端不可用，则应改为说明未能保存，不能报告为已记录。
