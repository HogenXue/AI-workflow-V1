## 工作流

1. 合并 `config/defaults.yaml` 与目标仓库的 `hogen-codex.yaml`，确认项目级发布要求。
2. 明确版本、变更范围、目标环境与授权边界，运行可用的构建、测试和兼容性检查。
3. 记录回滚触发条件和操作步骤；只有取得明确授权后才执行部署或灰度。
4. 发布工具、监控或生产访问不可用时，降级为本地构建、测试、版本文件、变更记录和只读部署配置检查。
