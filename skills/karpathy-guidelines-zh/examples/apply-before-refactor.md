# 重构前应用指南

## 用户请求

“把这个模块拆小一点，但别改行为。”

## 工作流

1. 按规则 1 列出假设：哪些文件在范围内、哪些测试证明行为不变。
2. 按规则 8 阅读 exports、调用方和现有测试，确认没有静默扩大范围。
3. 按规则 3 只做外科手术式移动；不顺手改命名或格式。
4. 按规则 4 定义成功标准：现有测试全过、对外 API 不变。
5. 每个重要步骤后用 [检查点模板](../templates/checkpoint.md) 汇报；无法描述当前状态时停止。

## 完成标准

行为未变且有测试或等价验证证据；若有未跑测试或未读模块，按规则 12 明确说明，不得声称已全部验证。
