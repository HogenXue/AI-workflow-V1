# Trellis 交接边界

```text
复杂 / 跨模块 / 需求不明确
  -> 取得创建或更新 Trellis task 的同意
  -> 检索仓库、当前 PRD、领域术语和 ADR
  -> Grill with Docs 作为 Phase 1.1 的唯一访谈器
  -> 每次一个问题；需求写 PRD，术语写 .trellis/spec/domain/，持久决定写 .trellis/spec/decisions/
  -> 用户确认 PRD
  -> Trellis 继续 Design / Plan / Research / Execute / Check / Finish
```

- 不得复制 Trellis PRD 为第二份 Spec 或 issue。
- 不得用领域/决定 spec 或访谈记录替代 Trellis task 状态。
- 简单且需求明确的任务跳过本 Skill。
- 本 Skill 和 `trellis-brainstorm` 是替代关系，不串联运行。
- 只有用户确认规划并授权实施后，Trellis 才能进入执行阶段。
