# 设计 API 变更

## 用户请求

“订单接口需要新增门店维度，并同步调整查询条件与数据库索引。请先给出可实施的规格。”

## 工作流

1. 合并 `config/defaults.yaml` 与项目根目录的 `hogen-codex.yaml`。
2. 按变更分类确认这是跨模块、接口和数据模型变更，必须创建规格。
3. 检测 OpenSpec CLI；可用时按项目约定创建或更新变更并校验，不可用时按变更规格模板落盘。
4. 写明 API 参数兼容性、迁移/回滚风险、索引影响和验证命令。

## 最终报告

已完成规格草案：订单查询新增门店维度，旧调用方保持默认行为；数据库迁移增加相应索引并准备回滚步骤。待实现前需确认历史订单的门店回填规则。若 OpenSpec CLI 不可用，应明确仅完成 Markdown 规格，未执行 CLI 校验。
